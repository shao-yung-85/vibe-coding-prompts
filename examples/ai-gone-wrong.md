# 🚨 翻車案例集：AI 亂改 → 哪個 prompt 救回來

真實的 vibe coding 翻車現場。每個案例:**發生什麼 → 為什麼會這樣 → 用哪個 prompt 救回來 → 學到什麼**。

> 這些情境你大概都遇過。重點不是「AI 很笨」,而是「換個下法就能避免」。

---

## 案例 1:我只想加一個按鈕,它重構了半個元件

**發生什麼:**
我說「在卡片右上角加一個刪除按鈕」,結果 AI 不只加了按鈕——它順手把整個 `Card` 元件改寫成它覺得「更乾淨」的寫法,改了 props 命名、拆了三個子元件,還動到別張卡片。我的其他功能當場壞了一片,diff 兩百多行根本沒法 review。

**為什麼會這樣:**
AI 天生愛「順便優化」。你給它一個小任務,它會把視野擴大到「整個檔案怎樣更好」,然後自作主張。

**救回來的 prompt:**(見 [護欄 #3 最小改動](../README.md#️-護欄-prompt不讓-ai-亂動))
```
用最小改動加這個刪除按鈕:只動必要的那幾行,不要重構、不要改 props 命名、
不要拆元件、不要動其他卡片。如果你覺得有什麼想改,先列出來問我。
```

**學到什麼:** 加東西時,預設要把 AI 的範圍「鎖死」。「最小改動」這四個字能擋掉八成的越界。

## 案例 2:重整頁面資料就消失,它一直亂加 useEffect

**發生什麼:**
待辦清單重新整理後就清空了。我說「修一下」,AI 加了一個 `useEffect`;沒好,它又加一個 `useEffect` 監聽別的東西;還是沒好,它開始改 state 結構……越改越亂,問題還在。

**為什麼會這樣:**
我給的是「修好它」,AI 只能用猜的——它不知道根因,就一個一個試,每次試都留下殘骸。

**救回來的 prompt:**(見 [Debug #1 先診斷](../README.md#-debug-除錯))
```
重整後待辦清單會清空。先不要改任何 code,先告訴我:
1. 這個現象代表什麼 2. 列出 2-3 個最可能的原因 3. 怎麼確認是哪一個
```
AI 立刻回:「最可能是沒把資料寫進 localStorage,或讀取時機在 render 之後。」一查就中,一行修好。

**學到什麼:** bug 不明時,**先診斷、禁止亂改**。讓 AI 提假設、給驗證方法,比讓它亂試快十倍。

## 案例 3:叫它寫測試,結果全綠卻什麼都沒測到

**發生什麼:**
我說「幫這個函式寫測試」,它寫了五個測試,全綠,我很安心。後來才發現:它全在測 happy path,而且有兩個測試根本是 `expect(true).toBe(true)` 等級的廢測試——邊界、錯誤輸入完全沒碰。我那個會在空字串爆掉的 bug,測試一個都沒抓到。

**為什麼會這樣:**
「寫測試」太模糊,AI 會選最容易讓測試變綠的寫法交差,而不是最能抓 bug 的寫法。

**救回來的 prompt:**(見 [寫測試 #1 涵蓋邊界](../README.md#-寫測試))
```
幫這個函式寫測試,務必涵蓋:邊界值(空字串、null、0、超長)、錯誤輸入(型別/格式錯)、
例外處理。每個測試用清楚的名稱說明它在驗證什麼。不要寫永遠會過的廢測試。
```

**學到什麼:** 測試要明確要求「測邊界和錯誤路徑」,否則綠燈只是假的安全感。

## 案例 4:它自作主張裝了 5 個套件

**發生什麼:**
請它加個日期格式化,它二話不說 `npm install` 了 `moment` + 三個相依,還改了 `package.json` 和鎖檔。我專案本來刻意保持零相依,這下被它塞了一堆肥套件,還是個已經不維護的。

**為什麼會這樣:**
AI 預設「裝個現成套件最快」,不會考慮你專案的相依策略——除非你講。

**救回來的 prompt:**(見 [護欄 #4 禁止破壞](../README.md#️-護欄-prompt不讓-ai-亂動) + [CLAUDE.md 禁止事項](../README.md#-claudemd-模板))
```
加一個日期格式化。規則:不要安裝任何新套件,用原生 Intl / Date 做。
如果你認為非裝不可,先停下來告訴我為什麼,由我決定。
```
更好的做法是直接寫進 `CLAUDE.md`:「不要安裝我沒同意的新套件。」一勞永逸。

**學到什麼:** 把「不准裝套件、不准改設定檔」這類紅線寫進 CLAUDE.md,就不用每次提防。

---

## 📌 共同的教訓

| 翻車原因 | 解法 |
|---------|------|
| AI「順便」優化越界 | 明令**最小改動**、鎖定檔案範圍 |
| 給「修好它」讓 AI 亂猜 | **先診斷**,要它提假設+驗證法再動手 |
| 模糊需求換來模糊結果 | 把「完成」拆成**具體、可驗收**的條件 |
| AI 自作主張(裝套件/改設定) | 把紅線寫進 **CLAUDE.md** 常駐護欄 |

一句話:**AI 不會讀心,但會照你給的邊界走。邊界給得清楚,翻車就少。**
